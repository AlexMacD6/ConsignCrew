# Remove Zip Code Validation from Edit Listing Page

## Problem
The edit listing page has zip code validation that is preventing users from saving changes. The "Save Changes" button is disabled when zip code validation fails, and the form submission is blocked.

## Current Issues
1. **Zip Code Validation State**: `zipCodeValid` state variable is blocking form submission
2. **Validation Function**: `validateZipCode` function is broken after partial removal
3. **useEffect Hook**: Zip code validation useEffect is still present and causing errors
4. **Form Submission**: `handleSubmit` function still references removed validation variables
5. **Button State**: Save Changes button is disabled based on validation state

## Required Changes

### 1. Remove Validation State Variables
```typescript
// Remove these lines:
const [zipCodeValid, setZipCodeValid] = useState<boolean | null>(null);
const [zipCodeValidating, setZipCodeValidating] = useState(false);
```

### 2. Remove Validation Function
```typescript
// Remove the entire validateZipCode function:
const validateZipCode = async (zip: string) => { ... }
```

### 3. Remove Validation useEffect
```typescript
// Remove the zip code validation useEffect:
useEffect(() => {
  if (zipCode && zipCode.length === 5) {
    validateZipCode(zipCode);
  } else if (zipCode && zipCode.length > 0) {
    setZipCodeValid(false);
  } else {
    setZipCodeValid(null);
  }
}, [zipCode]);
```

### 4. Update handleSubmit Function
```typescript
// Remove zip code validation from handleSubmit:
const handleSubmit = async (e: React.FormEvent) => {
  e.preventDefault();
  
  // Remove this validation block:
  // if (!zipCodeValid && zipCode) {

  //     setError("Please enter a valid zip code");
  //     return;
  //   }
  // }

  // Keep basic zip code check:
  if (!zipCode) {
    setError("Please enter a zip code");
    return;
  }
  
  // ... rest of function
};
```

### 5. Update Save Button
```typescript
// Change from:
disabled={saving || (zipCodeValid === false)}

// To:
disabled={saving}
```

### 6. Remove Validation UI Elements
```typescript
// Remove validation icons and messages from zip code input:
{zipCodeValidating && (
  <Loader2 className="absolute right-3 top-2.5 h-5 w-5 animate-spin text-gray-400" />

{zipCodeValid === true && (
  <CheckCircleIcon className="absolute right-3 top-2.5 h-5 w-5 text-green-500" />
```

## Expected Result
- Users can edit listings and save changes without zip code validation blocking them
- The Save Changes button is always enabled when not saving
- Form submission works normally
- No more linter errors related to zip code validation
