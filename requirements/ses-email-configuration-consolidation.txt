# SES Email Configuration Consolidation

## Overview
Consolidated AWS SES email configuration to use a single environment variable instead of duplicate configurations.

## Current State
The codebase is already using the consolidated `AWS_SES_DEFAULT_FROM_EMAIL` variable consistently across all files.

## Environment Variables to Use

### Required SES Variables
- `AWS_SES_DEFAULT_FROM_EMAIL=no-reply@treasurehub.club` - Primary sender email address
- `CONTACT_FORM_EMAIL=support@treasurehub.club` - Contact form notifications recipient
- `AWS_SES_CONFIGURATION_SET=treasurehub-deliverability` - SES configuration set for tracking

### Variables to Remove
- `SES_FROM_EMAIL` - This was the old variable name that should be removed

## Files Using SES Configuration
- `app/lib/ses-server.ts` - Main SES email sending utility
- `test-betterauth-setup.js` - SES configuration validation
- `test-betterauth-email.js` - Email testing utility

## Action Required
1. Update your `.env` file to remove the duplicate `SES_FROM_EMAIL` variable
2. Ensure only `AWS_SES_DEFAULT_FROM_EMAIL` is used
3. Verify all SES-related environment variables are properly set

## Recommended .env Configuration
```env
# AWS SES Email Configuration
AWS_SES_DEFAULT_FROM_EMAIL=no-reply@treasurehub.club
CONTACT_FORM_EMAIL=support@treasurehub.club
AWS_SES_CONFIGURATION_SET=treasurehub-deliverability
```

## Benefits
- Eliminates confusion from duplicate variables
- Consistent naming convention across the codebase
- Easier maintenance and configuration management
- Prevents potential conflicts between different email configurations
