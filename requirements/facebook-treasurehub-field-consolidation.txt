# Facebook-TreasureHub Field Consolidation Requirements

## Overview
Identify and consolidate duplicated fields between Facebook inputs and TreasureHub fields, ensuring Facebook-required formats are used where there is duplication.

## Duplicated Fields Analysis

### 1. Brand Field
- **TreasureHub Field**: `brand` (text input)
- **Facebook Field**: `facebookBrand` (text input)
- **Facebook Format**: Brand name as string
- **Consolidation**: Use Facebook brand field as primary, fallback to TreasureHub brand

### 2. Condition Field
- **TreasureHub Field**: `condition` (radio buttons: NEW, EXCELLENT, GOOD, FAIR)
- **Facebook Field**: `facebookCondition` (dropdown: new, used, refurbished)
- **Facebook Format**: new/used/refurbished (lowercase)
- **Consolidation**: Use Facebook condition field as primary, map TreasureHub condition to Facebook format

### 3. GTIN/UPC Field
- **TreasureHub Field**: `serialNumber` and `modelNumber` (separate text inputs)
- **Facebook Field**: `facebookGtin` (text input for GTIN/UPC)
- **Facebook Format**: 13-digit GTIN/UPC code
- **Consolidation**: Use Facebook GTIN field as primary, keep TreasureHub fields for internal use

## Implementation Requirements

### 1. Form Field Updates
- Remove duplicate Facebook-specific fields where they duplicate TreasureHub fields
- Update TreasureHub fields to use Facebook-required formats
- Maintain backward compatibility for existing data

### 2. Data Mapping
- Update condition mapping to use Facebook format (new/used/refurbished)
- Ensure brand field accepts Facebook-optimized brand names
- Map GTIN/UPC to appropriate Facebook field

### 3. API Integration
- Update Facebook shop feed to use consolidated fields
- Maintain fallback logic for existing data
- Ensure proper data transformation for Facebook API

### 4. Database Schema
- Consider removing duplicate Facebook fields where they're redundant
- Update field descriptions to reflect Facebook requirements
- Maintain data integrity during transition
