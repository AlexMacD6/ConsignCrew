# Facebook Shop Fields Display Integration

## Overview
Added comprehensive display and editing capabilities for Facebook Shop product specification fields in the list-item detail page and edit listing page.

## Changes Made

### 1. API Route Updates (`app/api/listings/[id]/route.ts`)
- Updated PUT endpoint to handle new Facebook Shop fields:
  - `facebookShopEnabled`, `facebookBrand`, `facebookCondition`, `facebookGtin`
  - Product specifications: `quantity`, `salePrice`, `salePriceEffectiveDate`, `itemGroupId`
  - Product attributes: `gender`, `color`, `size`, `ageGroup`, `material`, `pattern`, `style`, `productType`, `tags`

### 2. List-Item Detail Page (`app/(dashboard)/list-item/[id]/page.tsx`)
- Added data mapping for all new Facebook Shop fields in the transformed listing object
- Created new "Product Specifications" section displaying:
  - Color, Size, Gender, Age Group
  - Material, Pattern, Style, Product Type
  - Quantity Available, Sale Price
  - Tags (displayed as styled badges)
- Only shows the section when relevant fields have data
- Uses appropriate icons for each field type

### 3. Edit Listing Page (`app/(dashboard)/list-item/[id]/edit/page.tsx`)
- Added state management for all new Facebook Shop fields
- Created comprehensive "Product Specifications" form section with:
  - Quantity input (number, min 1)
  - Sale price with currency symbol
  - Sale price effective date (date picker)
  - Item group ID for product variants
  - Gender dropdown (male/female/unisex)
  - Age group dropdown (adult/kids/infant)
  - Color, Size, Material, Pattern, Style text inputs
  - Product Type text input
  - Tags system with add/remove functionality
- Added tag management functions:
  - `addTag()` - adds new tags
  - `removeTag()` - removes existing tags
  - `handleTagKeyPress()` - allows Enter key to add tags
- Updated form submission to include all new fields

## Field Descriptions

### Product Specifications
- **Quantity**: Number of items available for sale
- **Sale Price**: Special discounted price (optional)
- **Sale Price Effective Date**: When sale price becomes active
- **Item Group ID**: For grouping product variants together
- **Gender**: male, female, or unisex
- **Color**: Product color (e.g., Red, Blue, Black)
- **Size**: Product size (e.g., Large, Medium, 10)
- **Age Group**: adult, kids, or infant
- **Material**: Product material (e.g., Cotton, Leather, Wood)
- **Pattern**: Product pattern (e.g., Striped, Floral, Solid)
- **Style**: Product style (e.g., Modern, Vintage, Casual)
- **Product Type**: Specific product type (e.g., T-Shirt, Sofa, Smartphone)
- **Tags**: Array of relevant tags for better search visibility

## UI/UX Features
- Conditional display - sections only appear when relevant data exists
- Intuitive icons for each field type
- Tag system with visual badges and easy removal
- Responsive grid layout for form fields
- Proper validation and input types
